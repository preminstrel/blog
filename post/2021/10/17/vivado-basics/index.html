    在学习 FPGA 的过程中，我们需要用到 Vivado软件，这里我用的是 Vivado2021.1版本的软件。一般来说， FPGA 开发的流程分为以下几个部分来实现。

    graph LR id1(Create Project)-->id2(Code)-->id3(Complie)-->id4(Assign Pins)-->id5(Program Device)-->id6(Debug)-->id7(Program Flash) id6-.->id2

    Create Project

    Project Name

    可以新建一个工程，注意，location 中可以选择 create project subdirectory 来创建存放工程的文件夹，这一步的引导比较简单。

    Project Type

    选择 RTL Project

    Add Sources

    我们可以将之前已经写好的 Verilog 等文件加进工程中。

    Default Part

    选择合适的芯片型号

    Code

    这里用 Verilog 写代码，编辑器建议用 VS Code。设计了一个双路灯控电路

    写完代码后直接 Add Sources 添加文件即可。

    Compile

    设计完整个工程之后，要经过 Vivado 软件中几个工具的处理，分别是分析代码、综合电路以及生成目标芯片的实现内容。这些应用工具被聚集在一起，统称为编译器。

    graph LR id1(Run Synthesis)-->id2(Run Implementation)-->id3(Generate Bitstream)

    如果工程编译不通过，那么信息栏里就会提示报 Bug。建议查阅 Log 一栏下的报错信息，因为 Log 一栏下的信息打印是按顺序来进行的，那么在进行问题定位的时候就方便查找第一个报错的信息。一般来说， 往往后面顺序的信息很多都是由于第一个报错原因导致的，所以解决了第一个报错问题，基本上就可以同时解决其它的报错问题了。要注意的是，报错信息栏里的提示不会随时更新，它的信息是过时的，在解决了某些问题或者修改了代码后请重新编译更新信息状态栏。

    Assign Pins

    Vivado 需要先对工程进行编译再进行管脚配置，编译完后 ，在左侧工具栏中选 中 IMPLEMENTATION，然后选择 Open Implemented Design，在随后跳转的窗口右上角选择切换到 I/O Planning，开始管脚配置。

    配置完成后，保存管脚配置信息，命名管脚约束文件名后， 点击 OK 生成管脚约束文件。

    Program Device

    引脚分配完毕之后，需要重新进行一次全编译。当全编译通过后 Vivado 软件即可生成可以烧录到 FPGA 中的配置文件。将下载器 USB-Blaster 与 FPGA 开发板进行连接，打开 FPGA 开发板电源后回到 Vivado 主界面，在左侧工具栏 PROGRAM AND DEGUG 下展开 Open Hardware Manager，点击 Open Target，选择选项 Auto Connect，连接下载器。

    在左侧工具栏 PROGRAM AND DEGUG 下展开 Open Hardware Manager，点击 Program Device，选择相应的型号，打开下载界面。

    Debug

    下载完毕后即可进行电路的测试，我们可以利用 ILA 工具进行问题的定位和错误的排查。

    Program Flash

    1. 在 Vivado 软件里找到 Settings 设置选项，进入后点击 Bitstream 选项，将 bin_file 勾选。

    2. 点击 Generate Bitstream，生成 bit 文件和 bin 文件。

    3. 开发板上电，点击 Open Hardware Manager——Open Target——Auto Connect，连接板子。

    4. 在 Hardware 面板中右击 FPGA 器件，选择 Add Configuration Memery Device。

    5. 在弹出的添加配置存储器的界面中，找到板载的 Flash 存储器型号，完成添加。

    6. 找到二进制 bin 文件（project_name\project_name.runs\impl_1\xxx.bin），选中进行代码烧写，实现上电自启动，完成程序固化。

    7. 下载完成，断电重上电后运行的就是固化的程序。
