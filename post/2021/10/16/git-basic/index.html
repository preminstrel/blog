之前也是接触过，用过Git和GitHub，但是并没有完全系统地对它进行学习，所以借此机会，我想对Git进行一次系统全面的学习。

Installation

首先，安装Git，这里我们选择Scoop。

1

$ scoop install git

在git中，我们可以用Git Bash或者Git GUI来操控。当然，这里我选择了Windows Terminal。

Create Repository

Create a local repository

首先创建一个文件夹 mkdir work，然后进行 cd work ，对这个仓库进行初始化 。

1

$ git init

这样我们的仓库就创建好了。当然，前面的那些 git config 要自己配置一下。

Basic

git config --global user.email "preminstrel@gmail.com"

git config --global user.name "preminstrel"

git add：添加新文件

git commit -m "message"：提交change到branch上

git status：查看当前work tree的情况

git log：查阅Git的日志

git branch a：创建一个叫a的新分支

git checkout a：进入分支a

git branch -D a：删除分支a

git config --global http.proxy http://127.0.0.1:1081：设置代理

git config --global -l：查看设置

git config --global --unset http.proxy：取消代理

git config --global --unset https.proxy：取消代理

代理部分具体见这里

远程连接remote

创建SSH Key

因为本地Git仓库和GitHub仓库之间的传输是通过SSH加密传输的，GitHub需要识别是否是你推送，GitHub只要知道了你的公钥，就可以确认只有你自己才能推送，所以需要配置ssh key。

在用户主目录下，看看有没有.ssh文件，如果有，再看文件下有没有id_rsa和id_rsa.pub这两个文件，如果已经有了，可直接到下一步。如果没有，打开Git Bash，输入命令，创建SSH Key

1

$ ssh-keygen -t rsa -C "yourmail@gmail.com"

再去用户主目录里找到.ssh文件夹，里面有id_rsa和id_rsa.pub两个文件，这两个就是SSH Key的秘钥对，id_rsa是私钥，不能泄露，id_rsa.pub是公钥，可以公开。

Github上确认SSH Key

接下来到GitHub上，打开“Account settings”–“SSH Keys”页面，然后点击“Add SSH Key”，填上Title（随意写），在Key文本框里粘贴 id_rsa.pub文件里的全部内容。

验证

验证是否成功：

1

$ ssh -T git@github.com

Token

密码验证于2021年8月13日不再支持，也就是不能再用密码方式去提交代码。请用使用 personal access token 替代。

Rebase local Repository

有的时候需要rebase本地库，这个时候我们需要使用命令 git pull --rebase origin master

push不用输入密码-SSH传输

移除当前关联的远程仓库

1

git remote rm origin

添加新的ssh地址

1

git remote add origin ssh地址

4.再次提交

1

git push origin master
